# 工作流节点快速参考

## 🎯 核心节点

### ChatModel（大语言模型）
**配置项：**
- API配置：选择API端点
- AI平台：OpenRouter/OpenAI/Anthropic/Google/Meta
- 模型：智能搜索选择
- Temperature：0-2（精确↔创造）
- Max Tokens：100-8000

**使用场景：** 文本生成、对话、理解、分析

---

### Tool（工具调用）
**内置工具：**
- 🔍 网页搜索 (web_search)
- 🧮 计算器 (calculator)
- 💻 代码解释器 (code_interpreter)
- 📁 文件读取 (file_reader)
- 🎨 图片生成 (image_generator)
- 🕷️ 网页抓取 (web_scraper)
- 🗄️ 数据库查询 (database_query)
- 🌐 API调用 (api_caller)

**使用场景：** 扩展AI能力，执行特定任务

---

### Lambda（自定义函数）
**代码模板：**
1. 空模板
2. 文本处理
3. JSON解析
4. 数据过滤
5. API响应格式化
6. 条件判断

**使用场景：** 自定义逻辑、数据处理、格式转换

---

### Retriever（检索器）
**配置项：**
- 检索类型：向量/关键词/混合
- Top K：返回结果数量
- 数据源：知识库ID

**使用场景：** 知识库检索、RAG应用

---

## 🎯 逻辑控制节点

### Condition（条件判断）
**条件类型：**
- 表达式：`input.score > 0.8`
- 等于/大于/小于/包含

**使用场景：** 流程分支、条件路由

---

### Loop（循环）
**循环类型：**
- For循环
- While循环
- ForEach循环

**配置项：**
- 最大迭代次数
- 循环条件

**使用场景：** 批量处理、重复任务

---

### Switch（多路分支）
**配置项：**
- 判断字段
- 分支规则（JSON）

**使用场景：** 多条件路由、状态机

---

## 📊 数据处理节点

### Transform（数据转换）
**转换类型：**
- 映射 (map)
- 归约 (reduce)
- 过滤 (filter)
- 排序 (sort)

**使用场景：** 数据格式转换、结构调整

---

### Merge（数据合并）
**合并策略：**
- 连接 (concat)
- 合并 (merge)
- 覆盖 (override)
- 深度合并 (deep_merge)

**使用场景：** 多数据源合并

---

### Filter（数据过滤）
**配置项：**
- 过滤条件（函数表达式）

**使用场景：** 数据筛选、清洗

---

### Template（模板渲染）
**语法：** `{{variable}}`

**示例：**
```
Hello {{name}}, your score is {{score}}
```

**使用场景：** 动态文本生成、格式化输出

---

## 🔌 集成节点

### HTTP Request（HTTP请求）
**请求方法：**
- GET/POST/PUT/DELETE/PATCH

**配置项：**
- URL
- 请求头（JSON）
- 请求体（JSON）

**使用场景：** API调用、数据获取

---

### Webhook（Webhook触发器）
**认证方式：**
- 无
- Token
- Basic Auth
- API Key

**使用场景：** 外部事件触发、集成第三方服务

---

### Delay（延迟执行）
**配置项：**
- 延迟时间（毫秒）

**使用场景：** 流程控制、限流

---

## 📋 节点分类

| 分类 | 节点 |
|------|------|
| **核心** | ChatModel, Tool, Lambda, Retriever |
| **逻辑** | Condition, Loop, Switch |
| **数据** | Transform, Merge, Filter, Template |
| **集成** | HTTP Request, Webhook, Delay |

## 💡 快速提示

### ChatModel节点
- 创造性任务：Temperature 1.0-1.5
- 精确任务：Temperature 0.1-0.5

### Tool节点
- 优先使用内置工具
- 参数使用JSON格式

### Lambda节点
- 从模板开始
- 保持函数简洁
- 添加错误处理

### 工作流设计
- 使用有意义的节点名称
- 保持流程简洁
- 定期保存

---

## 🎯 常用工作流模式

### 1. 简单问答
```
Start → ChatModel → End
```

### 2. 工具增强
```
Start → ChatModel → Tool → ChatModel → End
```

### 3. 条件分支
```
Start → ChatModel → Condition
  ├─ True → Path A
  └─ False → Path B
```

### 4. 循环处理
```
Start → Loop → Lambda → End
```

### 5. API集成
```
Start → HTTP Request → Lambda → ChatModel → End
```
